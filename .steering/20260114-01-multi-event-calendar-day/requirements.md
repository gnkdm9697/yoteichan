# 要件定義: 同一日付の複数候補日対応

## 概要

カレンダーで日付を選択する際、同じ日付に複数の候補枠を作成できるようにする。

## 背景・課題

現状の動作:
- カレンダーの日付をクリック → トグル動作（選択/解除）
- 同じ日付は1つしか選択できない

ユーザーの課題:
- 同じ日に複数のイベント時間帯を設定したい
  - 例: 1/15 12:00〜13:00（ランチ）と 1/15 19:00〜21:00（飲み会）
- 現状では同じ日付を再度クリックすると削除されてしまう

## ユーザーストーリー

```
イベント主催者として、
同じ日付に複数の候補時間帯を設定したい。
なぜなら、1日の中で午前・午後・夜など複数の選択肢を提示したいから。
```

## 機能要件

### FR-1: カレンダー日付選択の動作変更

- カレンダーの日付をクリックした場合、常に新しい候補枠を追加する
- 既存の候補枠は削除しない（削除は候補日リストの×ボタンから行う）

### FR-2: 同一日付の複数候補対応

- 同じ日付を複数回選択可能にする
- 各候補枠は独立して時間・タイトルを設定できる
- 候補枠の識別はインデックスベースで行う

### FR-3: 日付順ソートの維持

- 候補日を追加した際、日付順でソートする
- 同じ日付が複数ある場合は追加順を維持

### FR-4: 編集画面での対応

- イベント編集画面（DateOptionEditor）でも同様の動作にする

## 非機能要件

- 既存のデータ構造を大きく変更しない
- バックエンドAPIの変更は不要（既にtitleフィールド対応済み）

## 受け入れ基準

- [ ] カレンダーで同じ日付を2回クリックすると、2つの候補枠が作成される
- [ ] 各候補枠に異なる時間・タイトルを設定できる
- [ ] 候補日リストの×ボタンで個別に削除できる
- [ ] 追加時は日付順でソートされる
- [ ] 編集画面でも同様に動作する
- [ ] ビルドが通る
