# 回答時の備考フィールド追加 - 要件定義

## 概要

回答者がイベントに出欠を回答する際、各候補日に対して備考（メモ）を入力できるようにする。

## ユーザーストーリー

**イベント回答者として**、各候補日に備考を追加したい。
**なぜなら**、「遅れます」「午後から参加」「早退予定」などの補足情報を主催者や他の参加者に伝えたいから。

## 機能要件

### 1. データモデル

- `responses` テーブルに `notes` カラム（TEXT型、NULL許可）を追加

### 2. 回答入力時

- 各候補日の○/△/×ボタンの近くに備考入力欄を表示
- 備考は任意入力（空でもOK）
- 最大100文字程度

### 3. 回答表示

- ResponseTableで各回答者の備考を表示
- 備考がある場合のみ表示

### 4. 回答編集時

- 既存の備考を編集可能

## 受け入れ基準

- [ ] 回答画面で各候補日に備考を入力できる
- [ ] 回答編集時に備考を変更できる
- [ ] 回答一覧で備考が表示される
- [ ] 備考が空の場合は何も表示されない

## 制約

- 既存の回答には影響を与えない（notes=NULLで動作）
- モバイルファーストのUI設計を維持
- 備考欄は目立ちすぎないシンプルなデザイン
