# 候補日への備考フィールド追加 - 要件定義

## 概要

候補日（date_options）に備考（notes）フィールドを追加し、各日付に対して自由記述のメモを入力できるようにする。

## ユーザーストーリー

**イベント主催者として**、各候補日に備考を追加したい。
**なぜなら**、日付ごとに補足情報（例：「この日は午前のみ可」「会場予約済み」など）を参加者に伝えたいから。

## 機能要件

### 1. データモデル

- `date_options` テーブルに `notes` カラム（TEXT型、NULL許可）を追加

### 2. イベント作成時

- 各候補日に備考入力欄を表示
- 備考は任意入力（空でもOK）
- 最大500文字

### 3. イベント編集時

- 既存の候補日の備考を編集可能
- 新規追加する候補日にも備考入力可能

### 4. イベント詳細表示（回答ページ）

- 候補日に備考がある場合、日付の下に小さく表示
- 回答者が備考を参照できる

## 受け入れ基準

- [ ] イベント作成画面で各候補日に備考を入力できる
- [ ] イベント編集画面で備考を編集できる
- [ ] 回答ページで備考が表示される
- [ ] 備考が空の場合は何も表示されない
- [ ] 500文字を超える入力は制限される

## 制約

- 既存のイベントには影響を与えない（notes=NULLで動作）
- モバイルファーストのUI設計を維持
